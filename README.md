Лабораторная работа №3: Автоматизированная работа с хранилищами данных с использованием фреймворка dbt (data build tool)
## 1. Цель работы
Целью данной работы является изучение принципов автоматизации процессов трансформации данных (ELT) внутри хранилища с использованием фреймворка dbt (data build tool).
##2. Развертывание фреймворка
Для реализации проекта было выбрано современное встраиваемое хранилище данных DuckDB. Процесс развертывания включал следующие этапы:
Активация виртуального окружения Python и установка адаптера командой pip install dbt-duckdb.
Инициализация проекта командой dbt init my_delivery_dbt, в ходе которой была создана структура папок для моделей, тестов и конфигураций.
Настройка профиля подключения в profiles.yml для работы с локальным файлом базы данных dev.duckdb.
## 3. Разработка скриптов автоматизированной работы (модели)
В соответствии с темой «Служба доставки еды», в папке models/ были разработаны SQL-скрипты, реализующие логику обработки данных:
Модель Staging (stg_food_orders.sql): Выполняет первичную обработку «сырых» данных.
Включает нормализацию названий ресторанов (перевод в верхний регистр) и приведение типов данных для идентификаторов и стоимости заказов.
Модель Marts (mart_restaurant_stats.sql): Формирует итоговую аналитическую витрину.
Используя функцию {{ ref() }} для связи моделей, скрипт рассчитывает ключевые показатели эффективности (KPI) для каждого ресторана: общее количество принятых заказов и суммарную выручку.
## 4. Развертывание и проверка
Запуск процесса трансформации и создание физических таблиц в базе данных DuckDB произведены с помощью команды: dbt run
Валидация данных в созданной витрине была выполнена через Python-интерфейс DuckDB, что подтвердило корректность расчетов агрегатов.
## 5. Постановка работы скриптов на расписание
Для обеспечения актуальности данных была настроена автоматизация запуска dbt без использования сторонних оркестраторов.
С помощью системного планировщика Cron в операционной системе Linux (WSL) была создана задача на ежечасное выполнение команды dbt run.
Проверка статуса планировщика через crontab -l подтвердила успешную регистрацию задачи:
0 * * * * cd ~/airflow_lab/my_delivery_dbt && ~/airflow_lab/airflow_env_final/bin/dbt run
## 6. Вывод
В ходе лабораторной работы был успешно развернут фреймворк dbt и реализован полный цикл ELT-процесса.
На примере данных службы доставки еды продемонстрирована возможность создания автоматизированных витрин данных, которые обновляются автономно по заданному расписанию.
Использование dbt позволило разделить логику хранения и логику трансформации данных, что является стандартом в индустрии Big Data.
